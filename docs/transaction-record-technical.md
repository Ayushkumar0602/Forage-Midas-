# Transaction record — Technical Documentation

## Overview
The `TransactionRecord` entity serves as the core persistence model for financial exchanges within the MidasCore system. It provides an immutable schema representation of a transfer between two parties, including the principal amount and any associated incentives applied to the transaction. This entity acts as the single source of truth for audit trails and balance reconciliation.

## Architecture
`TransactionRecord` resides in the persistence layer (`com.jpmc.midascore.entity`). It functions as a central node in the relational schema:
*   **Dependencies:** Relies on `UserRecord` to define the relationship endpoints (`sender` and `recipient`).
*   **Consumers:** It is consumed by `TransactionRepository` (typically an extension of `JpaRepository`) for CRUD operations and by service-layer components responsible for calculating account balances and processing ledger updates.

## Design Principles
*   **Domain-Driven Design (DDD):** The class represents a discrete domain event (a transaction). By strictly defining the relationship between `UserRecord` and the transaction, we enforce referential integrity at the database level.
*   **Encapsulation:** The internal state is protected through the use of private fields and lack of public setters, ensuring that once a transaction is instantiated, it remains immutable.
*   **Persistence-Ready:** The class follows the JPA/Jakarta Persistence specification, utilizing `@Entity` and `@ManyToOne` annotations to ensure the object-relational mapping (ORM) layer can effectively persist the object without manual mapping logic.

## API Reference

### `TransactionRecord`
*   **`TransactionRecord(UserRecord sender, UserRecord recipient, float amount, float incentive)`**
    *   Constructs a new record. Parameters are validated for non-null constraints by the JPA provider.
*   **`long getId()`**: Returns the primary key generated by the persistence provider.
*   **`UserRecord getSender()`**: Returns the origin `UserRecord` of the transaction.
*   **`UserRecord getRecipient()`**: Returns the target `UserRecord` of the transaction.
*   **`float getAmount()`**: Returns the principal amount transferred.
*   **`float getIncentive()`**: Returns the incentive value applied to this specific transaction.

## Internal Logic
The class relies on JPA's automatic primary key generation (`@GeneratedValue`). The entity uses a protected no-argument constructor, which is a requirement for Hibernate/JPA frameworks to instantiate proxy objects during lazy loading from the database. Logic flows primarily through the constructor, which mandates the inclusion of both involved parties and the financial values at the moment of object instantiation.

## Data Flow
1.  **Ingress:** Data enters via the constructor during the execution of a transfer service, usually after validating that the sender has sufficient funds.
2.  **Transformation:** The JPA provider translates the object state into an `INSERT` statement.
3.  **Egress:** When retrieved from the database, the `ManyToOne` relationships are managed by the persistence context, potentially triggering lazy loading of `UserRecord` entities if requested.

## Error Handling & Edge Cases
*   **Non-Nullable Constraints:** Fields are marked `nullable = false`. The JPA provider will throw a `PropertyValueException` if an attempt is made to persist a `TransactionRecord` with missing references (null `sender`, `recipient`) or uninitialized financial values.
*   **Referential Integrity:** Because the class maps to foreign keys in the database, the system will prevent the creation of transactions involving non-existent users, maintaining balance consistency.
*   **Precision:** As the amount and incentive fields use `float`, users of this class should be aware of potential floating-point rounding errors common in financial calculations; downstream service layers should handle rounding logic as necessary.

## Usage Example

### Persisting a New Transaction
```java
public void executeTransfer(UserRecord sender, UserRecord recipient, float amount, float incentive) {
    TransactionRecord transaction = new TransactionRecord(sender, recipient, amount, incentive);
    transactionRepository.save(transaction);
}
```

### Retrieving and Auditing
```java
TransactionRecord record = transactionRepository.findById(transactionId)
    .orElseThrow(() -> new EntityNotFoundException("Transaction not found"));

System.out.println("Sender: " + record.getSender().getName());
System.out.println("Total Transferred: " + (record.getAmount() + record.getIncentive()));
```
